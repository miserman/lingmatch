read.folderR Documentation

Process texts in a folder

Description

Read in and optionally segment all texts within a folder.

Usage

read.folder(path = NULL, segment = NULL, subdir = FALSE,
  ext = ".txt", fixed = TRUE, segment.size = NULL,
  bysentence = FALSE, reader = readLines, readarg = list(warn =
  FALSE), ncores = parallel::detectCores() - 2)

Arguments

path

Path to a folder containing files, or a vector of paths to files.

segment

Specifies how the text of each file should be segmented. If a number, texts will be broken into that many segments, each with a roughly equal number of words. If a character, texts will be broken at that character; for example, a string matching join will split texts on returns.

subdir

If TRUE files in folders in path will also be included.

ext

The extension of the files you want to read in. '.txt' by default.

fixed

If FALSE, and segment is a character, segment will be treated as a regular expression.

segment.size

If specified, segment will be ignored, and texts will be broken into segments containing roughly segment.size number of words.

bysentence

If TRUE, and segment is a number or wordcount is specified, sentences will be kept together, rather than being broken across segments.

reader

The function used to read files. File paths are always passed as the first argument. readLines by default.

readarg

A list of additional arguments to pass to reader, starting in the second position.

ncores

Number of CPU cores to use; defaults to total number of cores minus 2.
