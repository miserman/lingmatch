lma_lspaceR Documentation

Latent Semantic Space Operations

Description

Calculate and reduce the singular value decomposition of a dtm (i.e., create a latent semantic space), or map a dtm onto an existing latent semantic space.

Usage

lma_lspace(dtm, space, path = paste0(path.package("lingmatch"), "/data"),
  dim.cutoff = 0.5, keep.dim = FALSE)

Arguments

dtm

A matrix with terms as column names.

space

A matrix of right singular vectors (a latent semantic space), with terms as rownames. If missing, this will be calculated from the dtm. If a character, a file matching the character will be searched for in the lingmatch data folder (e.g., space = 'default'). If the file has a .sqlite extension, it will be only a subset will be loaded into RAM; this is slightly slower than using a full, preloaded .rda space, but faster than loading and unloading a complete .rda space, and less RAM intensive in all cases.

path

Path to a folder containing spaces. If not included, spaces will be looked for in the lingmatch data folder (paste0(path.package('lingmatch'),'/data')).

dim.cutoff

If a space is calculated, this will be used to decide on the number of dimentions to be retained: cumsum(d) / sum(d) < dim.cutoff, where d is a vector of singular values of dtm (i.e., svd(dtm)$d). The default is .5; lower cutoffs result in fewer dimensions.

keep.dim

Logical: if TRUE and a space is being calculated from the input, a matrix in the same dimentions as dtm is returned. Otherwise, a matrix with terms as rows and svd dimentions as columns is returned. The default, truncated matrix

Note

A general latent semantic space is a selection of right singluar vectors from the singular value decompositon of a dtm (svd(dtm)$v[,1:k], where k is the selected number of dimentions, decided here by cutoff).

Mapping a new dtm into a latent semantic space consists of multiplying common terms: dtm[,ct] %*% space[ct,], where ct = colnames(dtm)[colnames(dtm) %in% rownames(space)] – the terms common between the dtm and the space. This results in a matrix with documents as rows, and svd dimensions as columns, replacing terms.

Examples


text = c(
  "Hey, I like kittens. I think all kinds of cats really are just the best pet ever.",
  "Oh year? Well I really like cars. All the wheels and the turbos... I think that's the best
    ever.",
  "You know what? Poo on you. Cats, dogs, rabbits -- you know, living creatures... to think
     you'd care about anything else!",
  "You can stick to your opinion. You can be wrong if you want. You know what life's about?
    Supercharging, diesel guzzling, exhaust spewing, piston moving ignitions."
)

dtm = lma_dtm(text)

#using a space from a file might look something like this:
# lma_space(dtm,'default.zip','C:/user/name/downloads')

#calculate a latent semantic space from the example text
lss = lma_lspace(dtm)

#show that document similarities between the truncated and full space are the same
spaces = list(
  full = lma_lspace(dtm,keep.dim=TRUE),
  truncated = lma_lspace(dtm,lss)
)
sapply(spaces,lma_simets,metric='cosine')
